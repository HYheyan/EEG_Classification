# EEG 二分类实验报告：从 0.698 到 0.720 的完整探索

> 本报告记录了在 1694 样本 EEG 数据集上，针对二分类任务（background / target）进行的系统性实验。
> 共 22 个方向、315 次训练、11 组 5 种子稳定性评估。

---

## 1. 起点：基线架构与基线分数

### 1.1 原始 EEGNet 架构

```
输入 (B,1,59,282) → 多尺度时序卷积(4路, k=7/31/65/129)
  → 深度可分离空间坍塌(groups=f1) → BatchNorm → ELU → Pool → Dropout
  → 可分离时序卷积(k=17) → BatchNorm → ELU → Pool → Dropout
  → AdaptivePool → Flatten → FC → BN → ReLU → Dropout → FC(2)
```

- 参数量: ~27K (F1=16, D=2)
- 基线验证准确率: **0.6982**（AdamW + CosineAnnealing + BN）

### 1.2 训练配置

| 组件 | 配置 |
|------|------|
| 数据划分 | 分层 80/20 单次划分（快速筛选）/ 5 折 CV（最终评估） |
| 预处理 | CAR（共平均参考） + 逐样本 Z-score 归一化 |
| 数据增强 | 高斯噪声、时间平移、幅度缩放、通道 Dropout、时间掩码、频谱掩码、多尺度裁剪 |
| Batch Size | 128 |
| Epochs | 100（early stop patience=50）|

---

## 2. 核心改进：四项有效提升

### 2.1 RAdam + OneCycleLR

**改动**: 优化器从 AdamW 换成 RAdam（Rectified Adam），学习率调度从 CosineAnnealingWarmRestarts 换成 OneCycleLR。

**代码位置**: `train.py` — `_build_optimizer()` + 训练循环 `onecycle_scheduler.step()`

**效果**: 0.6982 → 0.7160 (+1.8%)，收敛从 epoch 86 提前到 epoch 34，快 40%。

### 2.2 TCN 膨胀卷积架构 (EEGNetTCN)

**问题**: 原始 EEGNet Block2 的单层时序卷积感受野仅 17 步（约 60ms），无法捕获更长程的 EEG 节律。

**方案**: 替换为 4 层堆叠的膨胀深度可分离卷积栈（Dilated Depthwise-Separable Conv Stack）：

```
原: Conv(k=17, dil=1), 感受野 = 17步
新: 4层 Conv(k=7, dil=[1,2,4,8]), 感受野 = 91步
    每层 = Depthwise(dil=d) + Pointwise + BN + 激活 + 残差 + Dropout
```

感受野计算: RF = 1 + 6×(1+2+4+8) = 91 步（~320ms），覆盖 delta/theta/alpha/beta 四个关键 EEG 频段。

**代码位置**: `model.py` — `EEGNetTCN` 类 + `_DilatedSepConv` 子模块

**效果**: 0.7160 → 0.7249 (+0.9%)，参数量仅增加 3K（30K 总数）。

### 2.3 ELU → LeakyReLU

**改动**: 将所有激活函数从 ELU 改为 LeakyReLU(negative_slope=0.1)。

**代码位置**: `model.py` — `_get_act()` 工厂函数

**效果**: 0.7249 → 0.7337 (+0.9%)，单点提升最大的一项改动。

**解释**: LeakyReLU 的负半轴线性响应（而非 ELU 的指数衰减）在 EEG 信号的小幅振荡区域保留了更多梯度信息。

### 2.4 SWA (Stochastic Weight Averaging)

**改动**: 从 epoch 20 开始维护参数移动平均，swa_lr=5e-4。

**效果**: 最高单次 0.7544（s42），但 7 种子均值 0.7164 ± 0.0272。SWA 对单次最优有明显提升，但引入高方差，其保守期望值（期望值 = 均值 - 0.5×标准差）实际低于基线。

## 3. 无效/负面改进：完整清单

以下所有方向均经过 5 种子稳定性验证或多次单种子测试，结论明确：

| 方向 | 结果 | 原因分析 |
|------|------|---------|
| SE 通道注意力 | 0.7160，无提升 | 空间坍塌后仅剩 1 个空间维度，通道注意力退化 |
| InstanceNorm | 0.6361，-6.2% | 破坏了电极间相对幅值信息 |
| GroupNorm | 0.7041，-2.0% | BN 的小批量统计噪声在此任务上可接受 |
| 被试嵌入 | 0.7219，无叠加 | TCN 已捕获足够时间模式，被试差异信息冗余 |
| TCN 加深 5 层 | 0.7189，-0.6% | 感受野 183 步过大，引入无关上下文 |
| TCN 加宽 F1=20 | 0.7012，-2.4% | 参数增加 → 小数据集过拟合 |
| Focal Loss (γ=1/2) | 历史负优化 | EEG 二分类难度分布不需要 focal 机制 |
| 类权重 (target ×1.5/2.0) | 0.6893 / 崩溃 | 加重 target 惩罚反而损害整体 |
| 平衡采样 | 0.7130，无提升 | 过采样 target 无效 |
| Bandpower 频带分支 | 0.7160 ± 0.025，与基线持平 | FFT 频带特征与 TCN 学习的特征冗余 |
| 熵正则 (0.01/0.02) | 0.7101 / 0.7124，均低于基线 | 置信度惩罚未改善泛化 |
| TTA (推理增强) | +0%，完全无效 | 单次推理已足够稳定 |
| Model Soup (权重融合) | 0.5325，毁模型 | 不同种子模型差异过大，直接平均失效 |
| 频谱增强 + 时间裁剪（训练增强） | 均值微降 | 过于激进的增强在 ~1700 样本上反效果 |

## 6. sub01 逐样本加权：一个实现 bug 的发现与修正

### 6.1 动机

测试集全是 sub01_sess3，而训练集含 sub01-05。全被试混合验证可能高估 sub01 的泛化能力。尝试在训练时对 sub01 样本的损失函数进行 2×/3×/4× 逐样本加权。

### 6.2 Bug 发现

**异常信号**: w=2.0 / 3.0 / 4.0 三组实验（各 5 种子，共 15 次训练）结果**完全一致**——均为 0.7201 ± 0.0306。

在独立随机种子下，三组不同超参数产生完全相同的统计量的概率极低。触发代码审查后，定位到两处实现缺陷：

**Bug 1 — 参数传递断裂** (`train.py:_build_params()`):
```python
# 修复前：_build_params() 返回的字典中缺少 "sub01_weight"
def _build_params() -> Dict:
    return {k: CONFIG[k] for k in [
        "model", "f1", ..., "entropy_reg",
        # ← "sub01_weight" 缺失！params.get("sub01_weight", 1.0) 永远回退 1.0
        "optimizer", ...
    ]}
```

**Bug 2 — 损失计算语义错误** (训练循环):
```python
# 修复前：CE loss 默认 reduction="mean" 返回标量
loss = criterion(logits, labels)          # 标量，shape=()
per_sample_w = 1.0 + (w - 1.0) * (subj==0).float()
loss = loss * per_sample_w.mean()         # 等价于 loss *= (1 + (w-1)*sub01_ratio)
# 不同 w 值之间仅差一个微小的乘法因子，被优化器自适应步长几乎完全抵消
```

**修复后**:
```python
# 用 F.cross_entropy(reduction="none") 得到逐样本 loss 向量
loss_vec = F.cross_entropy(logits, labels, weight=..., reduction="none")
sample_w = 1.0 + (w - 1.0) * (subj==0).float()
loss = (loss_vec * sample_w).mean()       # 真正的逐样本加权
```

### 6.3 修正后结果

| 权重 | 5种子均值 | 标准差 | 期望值 | 5 种子值 |
|------|----------|--------|--------|---------|
| w=2.0 | 0.7190 | **0.0086** | **0.7146** | 0.7308/0.7160/0.7249/0.7130/0.7101 |
| w=3.0 | 0.7183 | 0.0131 | 0.7118 | 0.7219/0.6982/0.7308/0.7130/0.7278 |
| w=4.0 | 0.7089 | 0.0168 | 0.7005 | 0.7160/0.7189/0.7219/0.6805/0.7071 |

**结论**: w=2.0 是所有单配置中标准差最低（0.0086）、期望值最高（0.7146）的方向。它不是"性能突破"——均值 0.7190 仍在稳定区间内——而是"稳定性改进"，大幅压缩了种子间方差。w 越高均值越低的剂量响应验证了实现正确性。

**教训**: 消融实验中，若某个参数的所有取值给出完全一致的结果，应先怀疑实现是否正确，而非直接宣布该参数无效。

## 7. 集成策略

### 7.1 5 折 CV 集成

5 个不同训练/验证划分的模型对同一样本取 softmax 平均。

- 均值: 0.7172，标准差: **0.0136**（所有评估方式中最低）
- 预测一致性: 80.8% 样本折间一致性 ≥80%，仅 9.1% 样本 ≤40%
- **最可靠的交付方案**

### 7.2 多配置集成（8 模型）

混合 SWA20×5 + wd=3e-3 + noSWA + Bandpower 共 8 个模型，预测分布: 355 target / 487 background。

集成的本质不是提升均值，而是用模型多样性对冲单个模型的随机 split 波动——属于推理侧方差控制策略。

## 9. 最终结论

### 9.1 性能上限

**单模型稳定上限约 0.717-0.720**，无法稳定接近 0.75。这由三个因素共同决定：

1. **样本量**: 1694 个样本，80/20 划分后训练集约 1355 个
2. **被试间差异**: 训练集含 5 个被试，某被试（sub04）准确率仅 65.62%
3. **目标类特征模糊**: target 类准确率 68.35% vs background 87.22%

要突破此上限，需要更多被试数据、被试内多 session 配对数据，或手工特征补充——而非更多超参数搜索。

### 9.2 11 组 5 种子稳定性排名（期望值降序）

| 排名 | 配置 | 均值 | 标准差 | 期望值 | 类型 |
|------|------|------|--------|--------|------|
| 1 | sub01_w2.0（修正后）| 0.7190 | 0.0086 | **0.7146** | 单配置-稳定性改进 |
| 2 | sub01_w3.0（修正后）| 0.7183 | 0.0131 | 0.7118 | 单配置 |
| 3 | 5 折 CV 集成 | 0.7172 | 0.0136 | 0.7104 | Ensemble-最可靠交付 |
| 4 | Pure lrelu (无 SWA) | 0.7179 | 0.0239 | 0.7060 | 单配置 |
| 5 | sub01_w4.0（修正后）| 0.7089 | 0.0168 | 0.7005 | 单配置 |
| 6 | Bandpower | 0.7160 | 0.0250 | 0.7035 | 单配置 |
| 7 | SWA20 基线 | 0.7164 | 0.0272 | 0.7028 | 单配置 |
| 8 | ENT 0.02 | 0.7124 | 0.0202 | 0.7023 | 单配置 |
| 9 | SWA25 | 0.7160 | 0.0286 | 0.7017 | 单配置 |
| 10 | dropout=0.55 | 0.7124 | 0.0238 | 0.7005 | 单配置 |
| 11 | ENT 0.01 | 0.7101 | 0.0262 | 0.6970 | 单配置 |

### 9.3 评估指标选择的教训

本实验使用 **期望值 = 均值 - 0.5×标准差** 作为排序依据。这个指标同时惩罚高方差和低均值。

- SWA20: 均值 0.7164，但标准差 0.0272 → 期望值 0.7028
- sub01_w2.0: 均值 0.7190，标准差 0.0086 → 期望值 0.7146

对 1694 样本的小数据集，单次最佳值（如 SWA20 s42 的 0.7544）本质上是噪声采样的上限。均值 + 方差才是方法本身的稳定属性。

### 9.4 交付策略

- **主力方案**: 5 折 CV 集成——最低方差，最可靠的评估指标
- **补充消融**: sub01_w2.0 单配置——最佳单配置稳定性，可作为报告亮点
- **推理侧**: argmax 即可，无需阈值校准或 TTA

## 11. 代码使用

    --cv kfold --k-folds 5 --epochs 100 --swa-start 20

# 训练 — sub01 加权（最佳单配置稳定性）
python train.py --model eegnettcn --act lrelu --optim radam --sched onecycle \
    --cv single --epochs 100 --mixup 0 --sub01-weight 2.0 --seed 42
